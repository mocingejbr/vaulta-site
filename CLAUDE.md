# Project: Vaulta Landing Page

## Goal
Create a modern, minimal landing page for Vaulta iOS app.
Must match app design language and feel premium, secure, and private.

## Brand Context
Vaulta is a privacy-first vault app.
Users store:
- photos
- videos
- documents
- files

Core values:
- privacy
- encryption
- control
- simplicity

## Design Language (IMPORTANT)
Follow iOS-style minimal design.

If design tokens provided:
- Use exact colors
- Use same font style (SF Pro or system font fallback)
- Match spacing and corner radius

If NOT provided:
- Use:
  - background: black / very dark (#0B0B0B)
  - primary: white
  - accent: subtle blue or purple
  - rounded corners (16–24px)
  - soft shadows
  - glass / blur effects (optional)

Style:
- clean
- minimal
- premium
- no clutter

## Layout

### Section 1 — Hero
- App name: Vaulta
- Tagline: "Your private vault. Fully encrypted."
- Short description:
  "Store photos, videos, and files securely. Only you have access."
- App Store download button (primary CTA)
- Optional: app mockup image

### Section 2 — Features
Show 4–6 key features:

- End-to-end encryption
- Hidden vault / decoy mode
- iCloud encrypted backup (premium)
- File + media support
- Break-in alerts
- Offline-first storage

Keep descriptions short (1 line each).

### Section 3 — How it works
Simple 3-step flow:

1. Import files
2. Everything encrypted
3. Unlock only with your passcode

### Section 4 — Trust / Security
- "Your data never leaves your control"
- "Encrypted before storage"
- "Recovery key required for restore"

Keep tone confident, not technical.

### Section 5 — CTA
Repeat App Store download button.

### Footer
- Privacy Policy link
- Terms & Conditions link
- App name
- Copyright

## Pages

Create 3 pages:

1. index.html (landing page)
2. privacy.html
3. terms.html

## Privacy Policy Page
Include:
- data collection (minimal or none)
- local storage explanation
- iCloud backup (encrypted)
- no access to user files
- contact email placeholder

## Terms Page
Include:
- usage terms
- subscription (premium features)
- no liability for lost recovery key
- account responsibility

## Tech Requirements
- Pure HTML + CSS (or minimal JS)
- Mobile-first responsive design
- Fast loading
- No heavy frameworks unless needed

## App Store Button
- Use official Apple badge style
- Link placeholder: #
- Make it prominent

## Constraints
- Keep text concise
- Avoid generic marketing fluff
- Focus on trust + privacy
- No tracking scripts

## Optional Enhancements
- subtle animations (fade-in)
- hover effects
- dark mode only (preferred)

## Output
- clean file structure
- readable HTML
- reusable CSS classes
- ready for GitHub Pages deploy
